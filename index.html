HTC Vive Tutorial voor Unity

De HTC Vive is een virtual reality headset gemaakt door HTC in samenwerking met Valve, bekend van Steam. Tijdens deze dojo gaan we zelf een game maken voor de HTC vive die je, als je goed doorwerkt, ook zelf kan spelen.

Als je al een keer met Unity hebt gewerkt heb je geluk. Ook deze keer gaan we weer met Unity aan de slag. Dankzij plugins van Valve voor Unity is het heel erg makkelijk om een VR game te maken voor de HTC vive.

Voor onze game gaan we de volgende stappen uitvoeren:

  • We vertalen de input van de controller naar de game
  • We zorgen er voor dat de objecten in de game kunnen reageren op de input.
  • We maken een laser waarmee we dingen kunnen aanwijzen.
  • En als laatste: We zorgen dat we kunnen teleporteren binnen de game.

Aan het eind van deze stappen heb je nu een sandbox waarin je kan experimenteren. Aan het eind van de tutorial zit nog een bonusopgave. Als je vroeg klaar bent kan je deze ook nog maken.

Waarschuwing: Iedereen reageert anders op beweging en rotatie als je een VR bril ophebt. Doe rustig aan als je straks je bril ophebt zodat je niet duizelig of misselijk wordt.

Omgaan met input

Neem een van de controllers in je hand en kijk er goed naar. Elke controller heeft de volgende inputs:

De touchpad lijkt een beetje op een joystick van een spelcomputer. De controller heeft ook een velocity (snelheid) en rotational velocity (draaisnelheid) terwijl je beweegt en draait; dit zal vooral handig blijken te zijn bij interactie met objecten in de virtuele wereld.

Tijd voor wat code! Maak een nieuw C#-script in de map Scripts, noem het ViveControllerInputTest, en open het in je favoriete IDE (bijvoorbeeld Visual Studio Code).

Verwijder de Start() methode en voeg boven de Update() methode het volgende toe:

// 1
private SteamVR_TrackedObject trackedObj;
// 2
private SteamVR_Controller.Device Controller
{
    get { return SteamVR_Controller.Input((int)trackedObj.index); }
}

Je hebt nu het volgende gemaakt:

  1. Een verwijzing naar het object dat wordt gevolgd. In dit geval een controller.
  2. Een Device eigenschap om gemakkelijke toegang tot de controller te bieden. Het gebruikt de index (verwijzing op de computer) van het gevolgde voorwerp om de invoer van de controller terug te geven.

Zowel het op het hoofd gemonteerde display en de controllers zijn tracked objects (gevolgde voorwerpen) - hun beweging en draaiing in de echte wereld worden gevolgd door de HTC Vive-basisstations en worden verzonden naar de virtuele wereld.

Voeg nu de volgende methode toe direcet boven de Update() methode:

void Awake()
{
    trackedObj = GetComponent<SteamVR_TrackedObject>();
}

Nadat dit script is geladen, krijgt het trackedObj (object dat wordt gevolgd) een verwijzing naar de SteamVR_TrackedObject -component die aan de controllers is gekoppeld:

Nu dat je toegang hebt tot de controller, kan je eenvoudig de invoer uitlezen. Voeg het volgende toe binnen de methode Update():

// 1
if (Controller.GetAxis() != Vector2.zero)
{
    Debug.Log(gameObject.name + Controller.GetAxis());
}

// 2
if (Controller.GetHairTriggerDown())
{
    Debug.Log(gameObject.name + " Trigger Press");
}

// 3
if (Controller.GetHairTriggerUp())
{
    Debug.Log(gameObject.name + " Trigger Release");
}

// 4
if (Controller.GetPressDown(SteamVR_Controller.ButtonMask.Grip))
{
    Debug.Log(gameObject.name + " Grip Press");
}

// 5
if (Controller.GetPressUp(SteamVR_Controller.ButtonMask.Grip))
{
    Debug.Log(gameObject.name + " Grip Release");
}

De bovenstaande code zorgt voor de meeste manieren waarop je toegang kan krijgen tot de invoer van de speler terwijl deze zich in VR bevindt. Het schrijft de naam van het GameObject naar de Console om eenvoudig onderscheid te maken tussen de linker en rechter controller. Hier is een beschrijving per onderdeel:

  1. Zoek de positie van de vinger wanneer deze zich op het touchpad bevindt en schrijf deze naar de Console.
  2. Als je de trigger inknijpt, schrijft deze ook een regel naar de Console. De trigger heeft speciale methoden om te controleren of deze is ingedrukt of niet: GetHairTrigger(), GetHairTriggerDown() en GetHairTriggerUp()
  3. Als je de trigger loslaat, schrijft het if-blok deze naar de Console.
  4. Als je op de Grip-knop drukt, schrijft dit stuk naar deze actie naar de Console. De GetPressDown() methode is de standaardmethode om te controleren of op een knop is gedrukt.
  5. Als je een van de gripknoppen loslaat, wordt deze actie naar de Console geschreven. Het gebruik van de methode GetPressUp() is de standaardmethode om te controleren of een knop is losgelaten.

Het script is nu klaar om te worden getest. Sla het op en ga terug naar de Unity-editor.

De controllers gebruiken met de Objecten

VR biedt gebruikers vele mogelijkheden die niet mogelijk zijn in de 'echte wereld', zoals het oppakken van voorwerpen, het onderzoeken en weggooien zonder achteraf op te ruimen.

Met HTC Vive kun je deze zorgeloze virtuele ervaring creren door enkele trigger colliders (daar waar je in de virtuele wereld met een object zou botsen) te gebruiken en een beetje code te schrijven.

Selecteer beide controllers in de Hierarchy en voeg een Rigidbody component eraan toe. (Add Component > Physics > Rigidbody)

Vink het selectievakje Is Kinematic en verwijder het vakje Use Gravity:

Voeg een Box Collider toe (Add Component > Physics > Box Collider) aan beide controllers en vink Is Trigger aan.

De standaard-collider is enorm, dus je zal het formaat moeten wijzigen en het verplaatsen. Stel Center in op (X:0, Y:-0.04, Z:0.02) en Size op (X:0.14, Y:0.07, Z:0.05). In dit geval heb je precies deze waarden nodig, omdat zelfs een een heel kleine afwijking een grote invloed heeft op waar de collider terechtkomt.

Run het spel opnieuw. Selecteer een controller in de Hierarchy en pak in de echte wereld controller op. Bekijk de Scene en concentreer je op de controller die je vasthoudt (druk op F). De collider gaat recht over het bovenste deel van de controller, het deel dat je gebruikt om voorwerpen op te pakken.

Zonder een script is deze collider weinig meer dan een nutteloze kubus - maak een nieuw C#-script in de map Scripts, noem het ControllerGrabObject en open het.

Verwijder de methode Start() en voeg deze (misschien bekende) code toe:

private SteamVR_TrackedObject trackedObj;

private SteamVR_Controller.Device Controller
{
    get { return SteamVR_Controller.Input((int)trackedObj.index); }
}

void Awake()
{
    trackedObj = GetComponent<SteamVR_TrackedObject>();
}

Dit is exact dezelfde code die je gebruikte voor het eerste script. Hiermee krijgt de controller een referentie en slaat het op voor later gebruik.

Voeg deze variabelen toe net onder de boventste trackedObj :

// 1
private GameObject collidingObject; 
// 2
private GameObject objectInHand;

Elke variabele heeft een doel:

  1. Slaat het GameObject op waar de trigger momenteel mee in botsing komt, dus je hebt de mogelijkheid om het object te pakken.
  2. Dient als een verwijzing naar het GameObject dat de speler momenteel aan het grijpen is.

Voeg dit toe onder de Awake() methode:

private void SetCollidingObject(Collider col)
{
    // 1
    if (collidingObject || !col.GetComponent<Rigidbody>())
    {
        return;
    }
    // 2
    collidingObject = col.gameObject;
}

Deze methode accepteert een collider (botser) als een parameter en gebruikt het GameObject als het collidingObject (botsende voorwerp) voor grijpen en loslaten. Bovendien is het:

  1. Maakt het GameObject niet tot een mogelijk grijpdoel als de speler al iets vasthoudt of als het voorwerp geen rigidbody (star object) heeft.
  2. Wijst het object toe als een potentieel grab-doel.

Voeg nu deze trigger-methoden toe:

// 1
public void OnTriggerEnter(Collider other)
{
    SetCollidingObject(other);
}

// 2
public void OnTriggerStay(Collider other)
{
    SetCollidingObject(other);
}

// 3
public void OnTriggerExit(Collider other)
{
    if (!collidingObject)
    {
        return;
    }

    collidingObject = null;
}

Deze methoden zorgen ervoor wat er zou moeten gebeuren wanneer de trigger-collider een andere collider betreedt en verlaat.

  1. Wanneer de trigger-collider op een andere botst, stelt dit de andere collider in als een mogelijk doel om te grijpen.
  2. Vergelijkbaar met sectie n (// 1), maar anders omdat het ervoor zorgt dat het doel wordt ingesteld wanneer de speler een controller een tijdje over een object heen houdt. Zonder dit kan de botsing mislukken of fout gaan.
  3. Wanneer de collider een object verlaat en een niet-gepakt doel verlaat, verwijdert deze code zijn doelwit door deze op null te zetten.

Vervolgens voeg je code toe om een object te pakken:

private void GrabObject()
{
    // 1
    objectInHand = collidingObject;
    collidingObject = null;
    // 2
    var joint = AddFixedJoint();
    joint.connectedBody = objectInHand.GetComponent<Rigidbody>();
}

// 3
private FixedJoint AddFixedJoint()
{
    FixedJoint fx = gameObject.AddComponent<FixedJoint>();
    fx.breakForce = 20000;
    fx.breakTorque = 20000;
    return fx;
}

In dit stuk code doe jij eigenlijk het volgende:

  1. Verplaats het GameObject in de hand van de speler en verwijder het van de collidingObject variabele.
  2. Voeg een nieuwe verbinding toe die de controller met het object verbindt met behulp van de methode AddFixedJoint() hieronder.
  3. Maak een nieuwe vaste verbinding, voeg deze toe aan de controller en stel deze zo in dat deze niet gemakkelijk breekt. Zo geef je het uiteindelijk terug.

Wat kan worden gepakt, moet ook weer worden losgelaten. Dit volgende blok code behandelt het loslaten van het object:

private void ReleaseObject()
{
    // 1
    if (GetComponent<FixedJoint>())
    {
        // 2
        GetComponent<FixedJoint>().connectedBody = null;
        Destroy(GetComponent<FixedJoint>());
        // 3
        objectInHand.GetComponent<Rigidbody>().velocity = Controller.velocity;
        objectInHand.GetComponent<Rigidbody>().angularVelocity = Controller.angularVelocity;
    }
    // 4
    objectInHand = null;
}

Deze code verwijdert de vaste verbinding van het vastgepakte object en regelt de snelheid en draaiing wanneer de speler het weggooit. De snelheid van de controller staat hier centraal. Zonder het te gebruiken, zou het losgelaten object recht naar beneden vallen, ongeacht hoe perfect je worp zou kunnen zijn. Vertrouw me, het voelt niet goed. :]

Omschrijving per stukje:

  1. Zorg ervoor dat er een vast verbinding aan de controller is bevestigd.
  2. Verwijder de connectie met het voorwerp dat door het verbinging wordt vastgehouden en vernietig de verbinding.
  3. Voeg de snelheid en draaiing van de controller toe wanneer de speler het object loslaat, zodat het resultaat een realistische boog is.
  4. Verwijder de verwijzing naar het eerder gekoppelde object.

Voeg ten slotte dit toe aan Update() om de controller input te verwerken:

// 1
if (Controller.GetHairTriggerDown())
{
    if (collidingObject)
    {
        GrabObject();
    }
}

// 2
if (Controller.GetHairTriggerUp())
{
    if (objectInHand)
    {
        ReleaseObject();
    }
}
  1. Wanneer de speler de trigger (trekker) indrukt en er een mogelijk grijpdoel is, grijpt hij deze.
  2. Als de speler de trigger loslaat en er een voorwerp aan de controller is bevestigd, wordt deze losgelaten.

Making A Laser Pointer

A laser pointer is handy in a VR world for all sorts of reasons. You can use them to pop virtual balloons, aim guns better and frustrate digital kittens.

Making one is quite simple. Youll just need a cube and another script. Start off by creating a new Cube in the Hierarchy (Create > 3D Object > Cube).

Name it Laser, set its position to (X:0, Y:5, Z:0), change the scale to (X:0.005, Y:0.005, Z:0) and remove the Box Collider component. Focus on it and you should see it floating above the rest of the level:

Lasers shouldnt cast shadows, and theyre always the same color, so you can get the desired effect by using an unlit material.

Create a new material in the Materials folder and name it Laser, and then change its shader to Unlit/Color and set its Main Color to pure red:

Assign the new material by dragging it onto the Laser in the Scene view. Alternatively, you can drag it onto the laser in the Hierarchy.

Finally, drag the Laser to the Prefabs folder and delete the original one from the Hierarchy.

Now make a new C# script named LaserPointer in the Scripts folder and open it. Add this familiar helper code:

private SteamVR_TrackedObject trackedObj;

private SteamVR_Controller.Device Controller
{
    get { return SteamVR_Controller.Input((int)trackedObj.index); }
}

void Awake()
{
    trackedObj = GetComponent<SteamVR_TrackedObject>();
}

Add these variables underneath trackedObj:

// 1
public GameObject laserPrefab;
// 2
private GameObject laser;
// 3
private Transform laserTransform;
// 4
private Vector3 hitPoint;
  1. This is a reference to the Lasers prefab.
  2. laser stores a reference to an instance of the laser.
  3. The transform component is stored for ease of use.
  4. This is the position where the laser hits.

Add this method to show the laser:

private void ShowLaser(RaycastHit hit)
{
    // 1
    laser.SetActive(true);
    // 2
    laserTransform.position = Vector3.Lerp(trackedObj.transform.position, hitPoint, .5f);
    // 3
    laserTransform.LookAt(hitPoint); 
    // 4
    laserTransform.localScale = new Vector3(laserTransform.localScale.x, laserTransform.localScale.y,
        hit.distance);
}

This method takes a RaycastHit as a parameter because it contains the position of the hit and the distance it traveled.

Stepping through each section:

  1. Show the laser.
  2. Position the laser between the controller and the point where the raycast hits. You use Lerp because you can give it two positions and the percent it should travel. If you pass it 0.5f, which is 50%, it returns the precise middle point.
  3. Point the laser at the position where the raycast hit.
  4. Scale the laser so it fits perfectly between the two positions.

Add the following inside the Update() method to make use of the players input:

// 1
if (Controller.GetPress(SteamVR_Controller.ButtonMask.Touchpad))
{
    RaycastHit hit;

    // 2
    if (Physics.Raycast(trackedObj.transform.position, transform.forward, out hit, 100))
    {
        hitPoint = hit.point;
        ShowLaser(hit);
    }
}
else // 3
{
    laser.SetActive(false);
}
  1. If the touchpad is held down
  2. Shoot a ray from the controller. If it hits something, make it store the point where it hit and show the laser.
  3. Hide the laser when the player released the touchpad.

Add the following inside of the empty Start() method:

// 1
laser = Instantiate(laserPrefab);
// 2
laserTransform = laser.transform;
  1. Spawn a new laser and save a reference to it in laser.
  2. Store the lasers transform component.

Save this script and return to the editor. Select both controllers in the Hierarchy and drag the laser script onto the Inspector to make it into a component.

Now drag the Laser prefab from the Prefabs folder onto the Laser slot in the Inspector:

Save your project and give the game another run. Pick up a controller, put on the headset and try holding the touchpad. Youll see a laser now:

Before moving on, remove the input test components from the controllers by right-clicking them and selecting Remove Component.

Youre removing them because they write strings for every frame and log them to the Console. Its not a great thing for performance, and every millisecond counts in VR. They were handy for testing the input, but should not be used for actual gameplay.

The next step is using this laser to teleport around the room!

Moving Around

Moving around in VR isnt as simple as pushing the player forward; doing so is a sure-fire way to induce nausea. A more feasible way to get around is teleportation.

The players sense of perception will accept a sudden position change more readily than a gradual one. Subtle changes in a VR setting can upset your feeling of balance and velocity more than suddenly finding yourself in a new place.

To show exactly where youll end up, youll use a marker or reticle thats provided in the Prefabs folder.
The reticle is a simple, unlit, circular disk:

To use the reticle, youll append the LaserPointer script, so open it in a code editor and add these variables to the top of the class:

// 1
public Transform cameraRigTransform; 
// 2
public GameObject teleportReticlePrefab;
// 3
private GameObject reticle;
// 4
private Transform teleportReticleTransform; 
// 5
public Transform headTransform; 
// 6
public Vector3 teleportReticleOffset; 
// 7
public LayerMask teleportMask; 
// 8
private bool shouldTeleport;

Each variable plays a role:

  1. Is the transform of [CameraRig].
  2. Stores a reference to the teleport reticle prefab.
  3. A reference to an instance of the reticle.
  4. Stores a reference to the teleport reticle transform for ease of use.
  5. Stores a reference to the players head (the camera).
  6. Is the reticle offset from the floor, so theres no Z-fighting with other surfaces.
  7. Is a layer mask to filter the areas on which teleports are allowed.
  8. Is set to true when a valid teleport location is found.

In the Update() method, replace this line:

if (Physics.Raycast(trackedObj.transform.position, transform.forward, out hit, 100))

With this one that takes the layer mask into account:

if (Physics.Raycast(trackedObj.transform.position, transform.forward, out hit, 100, teleportMask))

This makes sure the laser can only hit GameObjects that you can teleport to.

Also in the Update() method, add this code underneath the call to ShowLaser():

// 1
reticle.SetActive(true);
// 2
teleportReticleTransform.position = hitPoint + teleportReticleOffset;
// 3
shouldTeleport = true;

Heres what this does:

  1. Show the teleport reticle.
  2. Move the reticle to where the raycast hit with the addition of an offset to avoid Z-fighting.
  3. Set shouldTeleport to true to indicate the script found a valid position for teleporting.

While still in the Update() method, find laser.SetActive(false); and add the following line underneath it: 

reticle.SetActive(false);

This hides the reticle in the absence of a valid target.

Add the following method to handle the act of teleporting:

private void Teleport()
{
    // 1
    shouldTeleport = false;
    // 2
    reticle.SetActive(false);
    // 3
    Vector3 difference = cameraRigTransform.position - headTransform.position;
    // 4
    difference.y = 0;
    // 5
    cameraRigTransform.position = hitPoint + difference;
}

Who knew teleporting is as simple as five lines? Lets step through that code:

  1. Set the shouldTeleport flag to false when teleportation is in progress.
  2. Hide the reticle.
  3. Calculate the difference between the positions of the camera rigs center and the players head.
  4. Reset the y-position for the above difference to 0, because the calculation doesnt consider the vertical position of the players head.
  5. Move the camera rig to the position of the hit point and add the calculated difference. Without the difference, the player would teleport to an incorrect location. See the example below:

As you can see, the difference plays a crucial role in precisely positioning the camera rig and putting the player exactly where they expect to land.

Add this at the end of Update(), just outside the touchpad press if-else statement:

if (Controller.GetPressUp(SteamVR_Controller.ButtonMask.Touchpad) && shouldTeleport)
{
    Teleport();
}

This teleports the player if the touchpad is released and theres a valid teleport position.

Finally, add this code to the Start() method:

// 1
reticle = Instantiate(teleportReticlePrefab);
// 2
teleportReticleTransform = reticle.transform;
  1. Spawn a new reticle and save a reference to it in reticle.
  2. Store the reticles transform component.

Thats it! Save your script and return to Unity.

Select both controllers in the Hierarchy and take note of the new fields:

Drag [CameraRig] to the Camera Rig Transform slot, drag TeleportReticle from the Prefabs folder to the Teleport Reticle Transform slot and drag Camera (head) to the Head Transform slot.

Now set the Teleport Reticle Offset to (X:0, Y:0.05, Z:0) and set the Teleport Mask to CanTeleport. CanTeleport is not a default layer it was created for this tutorial. The Floor and Table objects are the only ones that are part of this layer.

Now play the game and use your laser on the floor to teleport around the room.

This sandbox is now fully functional and ready for endless tinkering!
